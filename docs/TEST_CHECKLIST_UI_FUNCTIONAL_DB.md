## Чек-лист тестирования интернет-магазина

Дата: 2026-03-18  
Окружение: Windows 10, Chrome (последняя версия), Node.js 20+, PostgreSQL 13+

### UI

- [ ] Главная страница открывается без визуальных артефактов
- [ ] Список товаров отображается карточками (изображение, название, цена)
- [ ] Фильтры (категория, бренд, размер, цвет) отображаются и визуально понятны
- [ ] Карточка товара корректно открывает страницу деталей
- [ ] Кнопки в карточках и в корзине имеют hover/active состояния
- [ ] Адаптивность: сайт корректно отображается на ширине 320px, 768px, 1440px
- [ ] Корзина показывает пустое состояние, если товаров нет
- [ ] Страница заказов показывает список заказов и статус

### Функционал

- [ ] Добавление товара (варианта размер+цвет) в корзину работает
- [ ] Повторное добавление того же варианта увеличивает quantity, а не дублирует строку
- [ ] Изменение количества товара в корзине пересчитывает итоговую сумму
- [ ] Удаление позиции из корзины удаляет ее из UI и БД
- [ ] Оформление заказа создает заказ и позиции заказа
- [ ] После оформления заказа корзина очищается
- [ ] При пустой корзине оформление заказа недоступно или возвращает корректную ошибку
- [ ] При недостаточном stock оформление заказа блокируется с понятным сообщением

### БД

- [ ] При добавлении в корзину создается запись в `carts` (если ее не было)
- [ ] При добавлении в корзину создается/обновляется запись в `cart_items`
- [ ] При изменении quantity обновляется `cart_items.quantity`
- [ ] При удалении позиции удаляется строка в `cart_items`
- [ ] При оформлении создается запись в `orders`
- [ ] При оформлении создаются записи в `order_items`
- [ ] При оформлении уменьшается `product_variants.stock`
- [ ] При ошибке insufficient stock транзакция откатывается (без частичных изменений)
