## Test plan: модуль “Корзина”

### 1. Цель

Проверить корректность работы модуля “Корзина” в MVP интернет‑магазина:

- добавление товара
- изменение количества
- удаление товара
- пересчёт суммы
- работа с повторными запросами
- базовая обработка ошибок

### 2. Объект тестирования

- Frontend: страницы `Каталог`, `Детали товара`, `Корзина`
- Backend: эндпоинты `/api/cart/*`, `/api/orders`
- БД: PostgreSQL таблицы `carts`, `cart_items`, `product_variants`, `orders`, `order_items`

### 3. Окружение

- ОС: Windows/Linux/macOS
- Браузер: Chrome/Firefox/Edge (последняя версия)
- Node.js: 18+ (рекомендуется 20+)
- PostgreSQL: 13+
- Данные: заполнены через `backend/seed.sql`

### 4. Предусловия

- Backend запущен и доступен `http://localhost:5000/api`
- Frontend запущен
- В БД есть товары и варианты (`products`, `product_variants`)
- В браузере localStorage доступен (для `sessionId`)

### 5. Набор тестов

#### TC-CART-01 — Добавление товара в корзину из карточки/деталей

- **Шаги**:
  - Открыть каталог
  - Открыть товар (или остаться на карточке), выбрать вариант
  - Нажать “Добавить в корзину”
  - Открыть корзину
- **Ожидаемо**:
  - В корзине появилась позиция с корректным названием/ценой/вариантом
  - Количество = 1
  - В БД появилась запись в `carts` (если не было) и `cart_items`

#### TC-CART-02 — Повторное добавление того же варианта

- **Шаги**:
  - Добавить один и тот же вариант два раза
  - Открыть корзину
- **Ожидаемо**:
  - Дубликатов строк нет
  - Количество увеличилось (например, стало 2)

#### TC-CART-03 — Изменение количества (увеличение)

- **Шаги**:
  - В корзине изменить quantity на 2/3/5
- **Ожидаемо**:
  - Количество обновилось
  - Сумма пересчиталась
  - В БД `cart_items.quantity` обновилось

#### TC-CART-04 — Изменение количества (некорректные значения)

- **Шаги**:
  - Попробовать ввести 0, отрицательное число, пустое значение
- **Ожидаемо**:
  - Количество не становится меньше 1 (сервер приводит к минимуму 1)
  - Ошибок/падений UI нет

#### TC-CART-05 — Удаление позиции

- **Шаги**:
  - Нажать “Удалить” у позиции
- **Ожидаемо**:
  - Позиция исчезла из UI
  - В БД запись из `cart_items` удалена
  - Сумма пересчиталась

#### TC-CART-06 — Подсчет суммы

- **Шаги**:
  - Добавить 2 разных товара
  - Поменять количество у одного
- **Ожидаемо**:
  - Итог = \(\sum price_i \* quantity_i\)

#### TC-CART-07 — Оформление заказа из корзины

- **Шаги**:
  - Имея товары в корзине, нажать “Оформить заказ”
- **Ожидаемо**:
  - Создана запись в `orders`
  - Созданы записи в `order_items`
  - `product_variants.stock` уменьшился на quantity
  - Корзина очищена (`cart_items` удалены для cart)

#### TC-CART-08 — Оформление заказа с недостаточным остатком

- **Шаги**:
  - Сделать так, чтобы в корзине quantity превышал stock (например, вручную обновить stock в БД)
  - Нажать “Оформить заказ”
- **Ожидаемо**:
  - Сервер вернёт `400 Not enough stock`
  - Заказ не создастся (нет новых строк в `orders/order_items`)
  - Остатки не изменятся

#### TC-CART-09 — Повторный запрос оформления заказа (idempotency-эффект)

- **Шаги**:
  - Оформить заказ
  - Повторить `POST /api/orders` для того же `sessionId`
- **Ожидаемо**:
  - Сервер вернёт `400 Cart is empty`
  - Нового заказа не создаётся

### 6. Критерии завершения

- Выполнены все тест‑кейсы
- Успешность ≥ 80%
- Нет критических дефектов, мешающих приёмке (каталог/корзина/заказ/БД)
